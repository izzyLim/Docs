LangGraph
사내 구축 가이드

BIP 프로젝트에서 LangGraph를 실전 적용한 경험을 기반으로, 사내 도입 판단부터 인프라 구성, 보안, 운영까지 전 과정을 안내합니다. 이미 동작하는 3개 에이전트의 실제 코드와 아키텍처를 참고할 수 있습니다.

BIP 실전 검증 Haiku + MCP Airflow 통합 PostgreSQL 월 ~$6 운영

도입이 필요한 상황 vs 불필요한 상황

필요
LangGraph를 써야 할 때
  • 에이전트가 결과를 보고 다시 시도해야 할 때 (루프)
  • 질문에 따라 다른 도구를 선택해야 할 때 (분기)
  • 사람이 중간에 승인/검토해야 할 때
  • 장시간 실행 중 크래시 복구가 필요할 때
  • 여러 에이전트가 협업해야 할 때
불필요
LangGraph 없이 충분할 때
  • 단순 질문 → LLM → 답변 (직선)
  • 문서 검색 → 요약 (RAG)
  • 정해진 순서 처리 (ETL)
  • → LangChain 또는 직접 구현이 더 단순

핵심 코드ReAct + MCP 패턴
from langgraph.prebuilt import create_react_agent
from langchain_mcp_adapters.client import MultiServerMCPClient

# MCP 도구 로드
mcp_client = MultiServerMCPClient({
    "bip-stock": {
        "url": "http://bip-stock-mcp:8000/sse",
        "transport": "sse"
    }
})
tools = await mcp_client.get_tools()  # 44개 도구

# ReAct 에이전트 생성
llm = ChatAnthropic(
    model="claude-haiku-4-5-20251001",
    max_tokens=4000
)
agent = create_react_agent(llm, tools, prompt=system_prompt)

# 실행
result = await agent.ainvoke({
    "messages": [{"role": "user", "content": checklist_text}]
})
💡
ReAct 패턴이 최적이었던 이유 Phase 2에서 룰 엔진으로 도구를 강제 매핑했지만, Phase 5에서 LLM ReAct로 전환. 새 체크리스트 항목에 코드 변경 없이 대응 가능해짐.

첫 에이전트: 가장 간단한 ReAct 패턴

PYTHON5분 만에 만드는 첫 에이전트
from langgraph.prebuilt import create_react_agent
from langchain_anthropic import ChatAnthropic
from langchain_core.tools import tool

# 1. 도구 정의
@tool
def search_db(query: str) -> str:
    """데이터베이스에서 검색"""
    return f"검색 결과: {query}"

@tool
def get_price(ticker: str) -> str:
    """종목 현재가 조회"""
    return f"{ticker}: 50,000원"

# 2. 에이전트 생성
llm = ChatAnthropic(model="claude-haiku-4-5-20251001")
agent = create_react_agent(
    llm,
    tools=[search_db, get_price],
    prompt="금융 데이터 분석 전문가입니다."
)

# 3. 실행
result = agent.invoke({
    "messages": [{"role": "user",
                  "content": "삼성전자 현재가 알려줘"}]
})

06Phase 3: 멀티 에이전트 + 웹 UI (2주)
PYTHONSupervisor 패턴 구현
from langgraph.graph import StateGraph, END
from typing import TypedDict, Annotated
import operator

class SupervisorState(TypedDict):
    messages: Annotated[list, operator.add]
    next_agent: str

def supervisor(state):
    response = llm.invoke(
        f"질문: {state['messages'][-1]}\n"
        f"선택지: data, analysis, news\n"
        f"에이전트 이름만 답해."
    )
    return {"next_agent": response.content.strip()}

workflow = StateGraph(SupervisorState)
workflow.add_node("supervisor", supervisor)
workflow.add_node("data", data_agent)
workflow.add_node("analysis", analysis_agent)
workflow.add_node("news", news_agent)

workflow.set_entry_point("supervisor")
workflow.add_conditional_edges(
    "supervisor", lambda s: s["next_agent"])
for agent in ["data", "analysis", "news"]:
    workflow.add_edge(agent, "supervisor")

app = workflow.compile(checkpointer=checkpointer)

09운영 팁 & BIP에서 얻은 교훈
흔한 문제
트러블슈팅
  • 무한 루프 — 반드시 max iteration
  • MCP 끊김 — try/finally로 close 보장
  • 토큰 폭발 — 긴 대화는 요약 또는 리셋
  • 스키마 변경 — 기존 체크포인트와 호환 주의
성능 최적화
효율화 팁
  • Haiku 우선 — 단순 판단용. Sonnet은 분석만
  • 도구 병렬 — asyncio.gather
  • 캐싱 — 자주 쓰는 데이터 Redis
  • 스트리밍 — invoke 대신 stream

BIP에서 얻은 교훈 5가지

교훈상세
프롬프트가 핵심LangGraph 구조보다 시스템 프롬프트 품질이 결과를 결정. 방향성 규칙(환율 higher_is_risk), 출력 형식, 금지 사항 등 명시적으로.
도구 선택은 LLM에Phase 2 룰 엔진 → Phase 5 LLM ReAct. 새 항목에 코드 변경 없이 대응.
하네스는 임시방편모델이 진화하면 우회 로직은 기술 부채. 항상 재검토 준비.
테스트 환경 분리테스트 DAG는 send_telegram=False. 운영 채널 오염 방지.
감사 로그 처음부터나중에 추가하면 기존 이력 없음. 에이전트와 동시에 구축.
