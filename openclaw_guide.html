OpenClaw
완전 가이드

OpenClaw은 LLM을 항상 켜진 자율 에이전트로 바꿔주는 오픈소스 프레임워크입니다. 개념부터 설치, 설정, 실제 사용까지 단계별로 정리했습니다.

01탄생 배경
배경어떻게 만들어졌는가

OpenClaw은 오스트리아 개발자 Peter Steinberger(PSPDFKit 창업자)가 만들었습니다. 그가 풀려던 문제는 단순했어요 — "WhatsApp으로 AI와 대화하고 싶다."

2025년 11월 론칭 후 빠르게 바이럴됐고, 2026년 1월 Anthropic과의 상표 분쟁으로 Clawdbot → Moltbot → OpenClaw으로 두 번 이름을 바꾸면서 오히려 더 유명해졌어요. 2026년 2월 Steinberger가 OpenAI에 합류하면서 현재는 독립 오픈소스 재단이 운영 중입니다.

핵심 아이디어는 "LLM은 두뇌, OpenClaw은 몸통" 이에요. 기존 챗봇이 탭을 닫으면 잊어버리는 것과 달리, OpenClaw은 항상 켜진 데몬으로 실행되면서 파일을 읽고, 명령어를 실행하고, 메시지를 보내는 실제 행동을 합니다.

02전체 아키텍처

OpenClaw은 5개 레이어로 구성됩니다. 각 레이어가 명확한 역할을 가지고 있어요.

OPENCLAW ARCHITECTURE
CHANNEL
채널 레이어
WhatsApp, Telegram, Slack, Discord, iMessage, WebChat, CLI — 사용자가 접근하는 모든 입구. 각 채널 어댑터가 메시지를 표준 형식으로 변환.
↓
GATEWAY
게이트웨이 — 중앙 제어판
항상 켜진 데몬. 모든 채널의 메시지를 받아 세션에 라우팅. WebSocket 서버(기본 127.0.0.1:18789). 도구 실행, 이벤트 처리, 메모리 관리를 조율.
↓
BRAIN
브레인 — LLM 엔진
Anthropic, OpenAI, Gemini, Ollama(로컬) 중 선택. 다단계 추론, 도구 호출 결정, 작업 분해. 모델을 교체해도 나머지 설정 유지.
↓
HANDS
핸즈 — 실행 레이어
파일 읽기/쓰기, 셸 명령어 실행, 브라우저 자동화, 웹 검색. OpenClaw을 실행하는 사용자와 동일한 권한으로 동작 — 강력하지만 위험.
↓
MEMORY
메모리 — 영구 기억
로컬 마크다운 파일로 대화 간 기억 유지. 선호, 연락처, 프로젝트, 학습 내용을 계층적으로 저장. 벡터 검색으로 관련 기억 검색.

메시지 처리 흐름 (7단계)

1. 채널 수신→ 2. 어댑터 변환→ 3. Gateway 라우팅→ 4. 세션 로드→ 5. LLM 추론→ 6. 도구 실행→ 7. 응답 전송
03Gateway — 중앙 제어판
개념Gateway란 무엇인가

Gateway는 OpenClaw의 핵심이에요. 모든 채널로부터 메시지를 받고, 에이전트 세션을 관리하고, LLM 호출을 조율하고, 도구를 실행하는 단일 프로세스입니다.

macOS에서는 launchd, Linux에서는 systemd 서비스로 등록되어 부팅 시 자동 시작되고 계속 실행 상태를 유지해요. 이것이 일반 챗봇과의 결정적 차이입니다 — 세션을 닫아도 에이전트는 계속 동작해요.

파일 구조~/.openclaw/ 디렉토리
~/.openclaw/
├── openclaw.json        ← 메인 설정 파일
├── auth-profiles.json   ← 모델 제공자 API 키
├── memory/
│   ├── preferences.md   ← 사용자 선호 및 습관
│   ├── contacts.md      ← 알게 된 사람들
│   ├── projects.md      ← 활성 프로젝트 컨텍스트
│   └── learnings.md     ← 에이전트가 학습한 내용
├── HEARTBEAT.md         ← 자율 작업 정의
└── workspace/           ← 에이전트 작업 공간
    ├── SOUL.md           ← 에이전트 성격 정의
    ├── AGENTS.md         ← 에이전트 행동 규칙
    └── USER.md           ← 사용자 정보
04Brain — LLM 엔진
개념Brain이란 무엇인가

Brain은 실제 추론을 담당하는 LLM이에요. OpenClaw은 모델에 종속되지 않아서 Anthropic, OpenAI, Google, 로컬 모델(Ollama) 중 원하는 것을 선택할 수 있어요.

OpenClaw은 Skills 목록을 메타데이터로만 LLM에 전달하고, LLM이 필요한 Skill을 요청하면 그때 전체 내용을 읽어줘요. 모든 Skill을 매번 프롬프트에 넣으면 토큰 낭비가 심해지기 때문이에요.

실전지원 모델 제공자
제공자추천 모델특징
Anthropicclaude-sonnet-4-6개발 작업에 가장 권장. 긴 컨텍스트, 코드 강점
OpenAIgpt-4o범용, 멀티모달
Googlegemini-2.0-flash빠른 응답, 비용 효율
Ollama (로컬)llama3, qwen2.5완전 로컬 실행, API 비용 없음
OpenRouter여러 모델 라우팅비용 최적화, 멀티 모델
05Hands — 도구 (Tools)
개념Hands란 무엇인가

Hands는 에이전트가 실제 세계와 상호작용하는 도구들이에요. 도구가 없으면 LLM은 텍스트만 생성할 수 있어요. 도구가 있어야 파일을 읽고, 명령어를 실행하고, 웹을 탐색할 수 있습니다.

중요: 도구는 OpenClaw을 실행하는 사용자와 동일한 권한으로 동작해요. exec 도구는 어떤 셸 명령어든 실행할 수 있어서 강력하지만, 잘못 사용하면 시스템 전체에 영향을 줄 수 있어요.

실전주요 도구 목록
도구기능위험도
read파일 읽기낮음
write파일 쓰기/수정중간
web_search웹 검색낮음
web_fetch웹 페이지 내용 읽기낮음
exec셸 명령어 실행높음 — 승인 모드 권장
browser브라우저 자동화 (클릭, 폼 작성)중간
process백그라운드 프로세스 관리중간
message채널로 메시지 전송낮음
cron예약 작업 설정중간
⚠
exec 도구 사용 시 필수 설정 exec 도구를 활성화한다면 반드시 승인 모드(exec.ask: "on")를 함께 설정하세요. 모든 명령어 실행 전 사용자에게 확인을 받습니다. 프롬프트 인젝션 공격이나 오판 시 마지막 방어선이에요.
06Memory — 영구 기억
개념Memory란 무엇인가

LLM은 본질적으로 무상태(stateless)예요. 세션이 끝나면 모든 것을 잊어버려요. OpenClaw은 로컬 마크다운 파일로 이 문제를 해결합니다.

대화 중 중요한 정보가 나오면 에이전트가 스스로 적절한 파일에 기록해요. 다음 세션이 시작되면 이 파일들을 읽어 맥락을 복원합니다. 벡터 임베딩으로 관련 기억을 의미 검색하는 것도 가능해요.

실전메모리 파일 구조
~/.openclaw/memory/
├── preferences.md   ← "사용자는 한국어 응답 선호, 다크 테마 좋아함"
├── contacts.md      ← "팀장: 홍길동, Slack ID: @gildong"
├── projects.md      ← "BIP-Pipeline: FastAPI + Oracle, 현재 NL2SQL 개발 중"
└── learnings.md     ← "Oracle에서 ROWNUM은 WHERE 절 안에 써야 함"
07Skills — 확장 모듈
개념Skills란 무엇인가

Skills는 OpenClaw의 확장 시스템이에요. YAML 프론트매터가 있는 마크다운 파일로, 에이전트에게 특정 도메인 작업 방법을 가르쳐줍니다.

도구(Tools)가 "할 수 있는 것"을 정의한다면, Skills는 "어떻게 잘 하는지"를 가르쳐요. Obsidian Skill을 설치하면 노트 정리 방법을 알게 되지만, write 도구가 없으면 실제로 파일을 쓸 수는 없어요. 도구와 스킬은 함께 동작합니다.

ClawHub에 5,400개 이상의 커뮤니티 스킬이 있지만, 보안 검토 없이 설치하면 위험할 수 있어요.

실전Skill 파일 형식
---
name: weather-briefing
trigger: "weather|forecast|temperature"
tools: [web-search, chat]
---

# Weather Briefing

When asked about weather, search for current conditions
and provide a concise briefing with temperature,
conditions, and any weather alerts.
⚡
ClawHub 스킬 보안 주의 커뮤니티 스킬 중 악의적인 것들이 발견된 사례가 있어요. 설치 전 반드시 소스 코드를 직접 확인하세요. Anthropic이 제공하는 공식 스킬이나 직접 만든 스킬을 우선 사용하는 것이 안전합니다.
08Heartbeat — 자율 실행
개념Heartbeat란 무엇인가

Heartbeat는 OpenClaw의 자율성을 만드는 핵심 기능이에요. 기본 30분마다 Gateway가 에이전트에게 하트비트 프롬프트를 보내요. 에이전트는 이 시점에 HEARTBEAT.md에 정의된 작업을 자율적으로 실행합니다.

예를 들어 "매일 오전 7시에 오늘의 일정 요약을 Telegram으로 보내줘"를 설정해두면, 사람이 요청하지 않아도 에이전트가 알아서 실행해요. 이게 단순 챗봇과의 결정적 차이입니다.

실전HEARTBEAT.md 예시
# 자율 작업 정의

## 매일 아침 브리핑
매일 오전 7시에:
1. Google Calendar에서 오늘 일정 확인
2. 중요 이메일 요약
3. Telegram으로 브리핑 전송

## 주간 리포트
매주 금요일 오후 5시에:
1. 이번 주 완료 작업 정리
2. 다음 주 준비사항 목록화
3. Slack #weekly 채널에 게시

11온보딩 — 초기 설정

온보딩은 설치의 핵심이에요. 건너뛰면 아무것도 동작하지 않아요. --install-daemon 플래그가 중요합니다 — 이것이 있어야 Gateway가 백그라운드 서비스로 등록돼요.

실전온보딩 실행
# 인터랙티브 방식 (권장 — 처음이라면)
openclaw onboard --install-daemon

# 자동화 방식 (API 키가 준비된 경우)
openclaw onboard --install-daemon \
  --auth-choice apiKey \
  --token-provider anthropic \
  --token "sk-ant-your-key-here"

온보딩 단계별 안내

1
모델 제공자 선택

Anthropic, OpenAI, Google, Ollama 중 선택. Anthropic Claude Sonnet 4.6 권장.

2
API 키 입력

선택한 제공자의 API 키를 붙여넣기. 건너뛰고 나중에 ~/.openclaw/openclaw.json에 직접 입력해도 돼요.

3
에이전트 이름과 성격 설정 (SOUL.md)

이름, 어떻게 상호작용할지 정의. SOUL.md가 생성돼요. 나중에 직접 수정 가능.

4
채널 연결 (선택)

Telegram, Discord, Slack 등 연결. 처음엔 WebChat(웹 UI)만으로 시작하는 걸 권장.

5
Gateway 시작 확인
# Gateway 상태 확인
openclaw gateway status

# 이 출력이 나오면 성공
✓ Gateway running on 127.0.0.1:18789
12주요 설정 — openclaw.json
실전핵심 설정 항목
~/.openclaw/openclaw.json (핵심 항목만 발췌) 
{
  "gateway": {
    "host": "127.0.0.1",       // 반드시 루프백으로 고정
    "port": 18789,
    "auth": {
      "mode": "token"          // 토큰 인증 필수
    }
  },
  "tools": {
    "exec": {
      "enabled": true,
      "ask": "on"             // 실행 전 승인 필수!
    },
    "read":  { "enabled": true },
    "write": { "enabled": true },
    "web_search": { "enabled": true }
  },
  "heartbeat": {
    "interval": 30             // 자율 실행 주기 (분)
  },
  "messages": {
    "language": "ko"           // 응답 언어
  }
}
13워크스페이스 파일

워크스페이스 파일들이 에이전트의 성격과 행동을 정의해요. 모두 마크다운이라 직접 수정하기 쉬워요.

실전주요 워크스페이스 파일 설명

SOUL.md — 에이전트 성격

# SOUL.md

이름: Azi
성격: 간결하고 정확함. 불필요한 말 없이 핵심만.
언어: 한국어로 응답
전문성: 데이터 엔지니어링, Python, SQL
톤: 전문적이지만 친근하게

AGENTS.md — 행동 규칙

# AGENTS.md

## 규칙
- .env 파일 내용 절대 출력 금지
- 파괴적 작업 전 반드시 확인
- 불확실하면 추측하지 말고 물어볼 것
- 코드 변경 후 항상 변경 파일 목록 요약

USER.md — 사용자 정보

# USER.md

이름: Alex
직책: 데이터 엔지니어
주요 프로젝트: my-project (FastAPI + PostgreSQL)

18보안 설정 — 필수
⚠
OpenClaw은 높은 권한의 장기 실행 서비스입니다 일반 챗봇과 다르게 파일 시스템 접근, 명령어 실행, 브라우저 자동화까지 가능해요. 보안 설정을 소홀히 하면 공격 표면이 매우 넓어집니다.
필수 보안 설정최소한 이것은 해야 해요

1. Gateway는 반드시 로컬호스트만

{
  "gateway": {
    "host": "127.0.0.1"   // 절대 0.0.0.0 으로 바꾸지 마세요
  }
}

2. exec 도구에 승인 모드 필수

{
  "tools": {
    "exec": {
      "enabled": true,
      "ask": "on"    // 모든 명령 실행 전 사용자 확인
    }
  }
}

3. 민감 디렉토리 접근 제한

{
  "tools": {
    "read": {
      "deny": [
        "~/.ssh",
        "~/.aws",
        "**/.env",
        "**/*.pem",
        "**/*.key"
      ]
    }
  }
}

21멀티에이전트 패턴

OpenClaw은 하나의 Gateway 안에서 여러 에이전트를 독립적으로 운영하거나, 에이전트가 서브에이전트를 동적으로 생성하는 오케스트레이터 패턴을 지원해요.

언제멀티에이전트가 필요한 경우
  • 도구 격리 — 업무용 에이전트는 exec 권한이 필요하지만 공개 Discord 봇은 그러면 안 될 때
  • 병렬 처리 — 리서치, 코드 작성, 테스트를 동시에 실행하고 싶을 때
  • 메모리 격리 — 업무 에이전트와 개인 에이전트의 기억이 섞이지 않아야 할 때
  • 모델 비용 최적화 — 오케스트레이터는 비싼 모델, 단순 작업 서브에이전트는 저렴한 모델 사용
⚡
멀티에이전트는 복잡도가 올라가요. 단일 에이전트로 해결되면 단일 에이전트를 쓰세요. 에이전트를 추가하는 명확한 이유가 있을 때만 추가하세요.

패턴 1 — 영구 독립 에이전트 (채널별 라우팅)

설정각 채널마다 다른 에이전트 연결
// openclaw.json
{
  "agents": {
    "list": [
      {
        "id": "work",
        "name": "업무 에이전트",
        "model": "anthropic/claude-sonnet-4-6",
        "workspace": "~/.openclaw/agents/work",
        "tools": { "allow": ["exec", "read", "write"] }
      },
      {
        "id": "public",
        "name": "공개 봇",
        "model": "anthropic/claude-haiku-4-5",
        "workspace": "~/.openclaw/agents/public",
        "tools": { "allow": ["web_search"] }  // exec 없음!
      }
    ],
    "bindings": [
      { "channel": "telegram", "agent": "work" },
      { "channel": "discord", "agent": "public" }
    ]
  }
}

에이전트 관리 명령어

# 에이전트 목록 확인
openclaw agents list

# 바인딩 확인
openclaw agents list --bindings

# 에이전트 추가
openclaw agents add

패턴 2 — 서브에이전트 (동적 생성)

개념오케스트레이터 → 서브에이전트 구조

메인 에이전트가 작업 중 무거운 서브태스크를 서브에이전트에게 위임해요. 서브에이전트는 비동기로 실행되고 완료되면 결과를 부모에게 전달해요.

# 채팅 중 서브에이전트 수동 생성
/subagents spawn --prompt "경쟁사 제품 3개 가격 조사해서 표로 정리해줘" \
  --model anthropic/claude-haiku-4-5

# 서브에이전트 상태 확인
/subagents list

# 서브에이전트 중단
/subagents kill <id>

openclaw.json에서 서브에이전트 모델 설정

{
  "agents": {
    "defaults": {
      "subagents": {
        "model": "anthropic/claude-haiku-4-5", // 저렴한 모델로
        "maxSpawnDepth": 2,  // 오케스트레이터 패턴 허용
        "archiveAfterMinutes": 60
      }
    }
  }
}
ℹ
서브에이전트 비용 주의 서브에이전트는 각각 독립적인 컨텍스트와 토큰 사용량을 가져요. 무거운 작업이나 반복 작업에는 저렴한 모델을 서브에이전트에 설정하면 비용을 크게 줄일 수 있어요.

증상별 해결법

오류 1openclaw: command not found

설치는 됐는데 명령어를 못 찾는 경우. npm global bin이 PATH에 없는 거예요.

# 해결 — ~/.zshrc 또는 ~/.bashrc에 추가
export PATH="$(npm prefix -g)/bin:$PATH"

# 적용
source ~/.zshrc

# 확인
which openclaw
오류 2Gateway start blocked: set gateway.mode=local

가장 흔한 오류. Gateway 모드가 설정되지 않으면 보안상 시작을 거부해요.

# 해결
openclaw config set gateway.mode local
openclaw gateway restart
오류 3EADDRINUSE — 포트 18789 충돌

다른 프로세스가 포트를 점유하고 있어요. 구버전(Clawdbot) 서비스가 남아있는 경우가 많아요.

# 포트 점유 프로세스 확인
lsof -i :18789

# 구버전 서비스 제거 (Linux)
systemctl --user stop clawdbot-gateway.service
systemctl --user disable clawdbot-gateway.service

# 포트 변경으로 해결
openclaw config set gateway.port 18790
openclaw gateway restart
오류 4gateway token missing / unauthorized

WebChat UI 접속 시 토큰이 없거나 만료된 경우예요.

# 현재 토큰 확인
openclaw config get gateway.auth.token

# 새 토큰 생성
openclaw doctor --generate-gateway-token

# 토큰을 고정해서 재시작마다 바뀌지 않게
openclaw config set gateway.token "your-stable-token"

# 모델 연결 상태 확인

오류 7Gateway가 서비스로 안 켜짐 (재부팅 후)

데몬 등록은 됐는데 부팅 후 자동 시작이 안 되는 경우. 환경변수가 서비스에 전달이 안 되는 게 주요 원인이에요.

# macOS — launchd 재등록
openclaw gateway install --force
openclaw gateway restart

# Linux — systemd 상태 확인
systemctl --user status openclaw-gateway.service
journalctl --user -u openclaw-gateway.service -n 50
