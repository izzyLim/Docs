1. LangGraph란?

한 줄 정의

AI 에이전트 워크플로우를 그래프(노드 + 엣지)로 모델링하는 상태 기반 오케스트레이션 프레임워크. LangChain 위에 구축되었지만 독립적으로도 사용 가능.

왜 LangGraph인가? — LangChain과의 차이

항목LangChain (Chain)LangGraph (Graph)
실행 흐름A → B → C (선형)분기, 루프, 병렬 가능
상태 관리없음 (입출력 전달만)공유 State 객체
Human-in-the-Loop수동 구현네이티브 interrupt()
내구성없음체크포인터로 크래시 복구
멀티 에이전트어색한 통합1등 시민 설계
적합 상황단순 파이프라인, RAG복잡한 에이전트, 루프/분기 필요
판단 기준: 워크플로우에 분기(if-then)나 루프(retry/revise)가 필요하면 LangGraph, 단순 직선 파이프라인이면 LangChain으로 충분.

1) State (상태)

그래프 전체에서 공유되는 메모리 객체. 모든 노드가 읽고 쓴다.

class AgentState(TypedDict):
    messages: List[str]    # 대화 이력
    research: str          # 리서치 결과
    draft: str             # 작성된 초안
    iteration: int         # 반복 횟수

Channel Reducer로 병합 전략 정의 가능 (배열은 concat, 스칼라는 override)

2) Node (노드)

하나의 작업 단위 (함수). State를 받아서 변환 후 반환.

def research_agent(state: AgentState):
    result = llm.invoke(state["messages"])
    return {"research": result.content}

각 노드는 독립적으로 테스트 가능. async 함수도 지원.

3) Edge (엣지)

노드 간 직접 연결. A가 끝나면 B로 이동.

workflow.add_edge("researcher", "writer")
# researcher 완료 → writer 실행

4) Conditional Edge (조건 분기)

조건에 따라 다른 노드로 라우팅. 루프의 핵심.

def should_revise(state):
    if state["fact_check"] == "FAIL" \
       and state["iteration"] < 3:
        return "writer"  # 다시 작성
    return "formatter"   # 완료

workflow.add_conditional_edges(
    "fact_checker", should_revise,
    {"writer": "writer",
     "formatter": "formatter"}
)

5) Checkpointer (체크포인터)

매 노드 실행 후 상태 자동 저장. 크래시 복구, 대화 지속.

# 개발용
checkpointer = MemorySaver()

# 프로덕션용
checkpointer = PostgresSaver.from_conn_string(
    "postgresql://user:pass@host/db"
)

app = workflow.compile(
    checkpointer=checkpointer
)
유형용도특징
MemorySaver개발/테스트인메모리, 재시작 시 소멸
SqliteSaver경량 프로덕션파일 기반, 단일 프로세스
PostgresSaver엔터프라이즈수평 확장, 다중 인스턴스

6) Human-in-the-Loop (사람 개입)

특정 노드 전에 실행 일시 중지. 사람이 검토/승인 후 재개.

# 방법 1: 특정 노드 전 중단
app = workflow.compile(
    interrupt_before=["fact_checker"]
)

# 방법 2: 노드 안에서 직접 중단
def review_node(state):
    decision = interrupt({
        "message": "검토해주세요",
        "plan": state["draft"],
        "actions": ["approve", "reject"]
    })
    return {"approved": decision == "approve"}

승인 게이트, 콘텐츠 검수, 컴플라이언스 워크플로우에 필수

3. 그래프 구축 전체 흐름

sequenceDiagram participant D as 개발자 participant SG as StateGraph participant N as 노드 등록 participant E as 엣지 연결 participant C as 컴파일 participant R as 실행 D->>SG: StateGraph(AgentState) 생성 D->>N: add_node("name", function) D->>N: add_node("name2", function2) D->>E: set_entry_point("name") D->>E: add_edge / add_conditional_edges D->>C: workflow.compile(checkpointer=...) C-->>R: 실행 가능한 app 반환 R->>R: app.invoke(initial_state) R->>R: 또는 app.stream(initial_state)

완전한 코드 예시: 리서치 → 작성 → 팩트체크 루프

from langgraph.graph import StateGraph, END
from typing import TypedDict, List

# 1. State 정의
class AgentState(TypedDict):
    messages: List[str]
    research: str
    draft: str
    fact_check_result: str
    iteration: int

# 2. 노드 함수
def researcher(state: AgentState):
    result = llm.invoke(f"조사해줘: {state['messages'][-1]}")
    return {"research": result.content}

def writer(state: AgentState):
    result = llm.invoke(f"이 자료로 글 써줘: {state['research']}")
    return {"draft": result.content, "iteration": state["iteration"] + 1}

def fact_checker(state: AgentState):
    result = llm.invoke(f"팩트체크: {state['draft']}")
    return {"fact_check_result": result.content}

# 3. 조건 분기
def should_revise(state: AgentState):
    if "FAIL" in state["fact_check_result"] and state["iteration"] < 3:
        return "writer"    # 다시 작성
    return END             # 완료

# 4. 그래프 구성
workflow = StateGraph(AgentState)
workflow.add_node("researcher", researcher)
workflow.add_node("writer", writer)
workflow.add_node("fact_checker", fact_checker)

workflow.set_entry_point("researcher")
workflow.add_edge("researcher", "writer")
workflow.add_edge("writer", "fact_checker")
workflow.add_conditional_edges("fact_checker", should_revise)

# 5. 컴파일 + 실행
app = workflow.compile()
result = app.invoke({
    "messages": ["삼성전자 반도체 사업 현황 분석해줘"],
    "research": "", "draft": "", "fact_check_result": "",
    "iteration": 0,
})

Supervisor 구현 코드

def supervisor(state: AgentState):
    """질문을 분석하고 적절한 에이전트로 라우팅"""
    response = llm.invoke(
        f"이 질문을 어떤 에이전트가 처리해야 할까?\n"
        f"질문: {state['messages'][-1]}\n"
        f"선택지: coder, researcher, data_analyst\n"
        f"에이전트 이름만 답해."
    )
    return {"next_agent": response.content.strip()}

def route_to_agent(state: AgentState):
    return state["next_agent"]

workflow.add_conditional_edges(
    "supervisor", route_to_agent,
    {
        "coder": "coder",
        "researcher": "researcher",
        "data_analyst": "data_analyst",
    }
)

Sub-graph (하위 그래프)

복잡한 에이전트를 독립된 하위 그래프로 분리. 각각 자체 State 스키마와 체크포인팅.

# 팀 A의 하위 그래프
team_a = StateGraph(TeamAState)
team_a.add_node("worker_a1", worker_a1_fn)
team_a.add_node("worker_a2", worker_a2_fn)
team_a_compiled = team_a.compile()

# 메인 그래프에 하위 그래프 추가
main_workflow.add_node("team_a", team_a_compiled)

Channel Reducer: 상태 병합 전략

데이터 타입Reducer동작예시
메시지 리스트 operator.add 기존 리스트에 추가 (append) ["A"] + ["B"] = ["A","B"]
단일 값 기본 (덮어쓰기) 최신 값으로 교체 "old" → "new"
카운터 커스텀 함수 누적 lambda old, new: old + new 
from typing import Annotated
import operator

class AgentState(TypedDict):
    # 메시지는 누적 (append)
    messages: Annotated[list, operator.add]
    # 반복 횟수는 덮어쓰기
    iteration: int
    # 에이전트 이름은 덮어쓰기
    current_agent: str

State 설계 베스트 프랙티스

  • 플랫 dict 피하기 — 에이전트별 네임스페이스 분리 (state.research, state.draft)
  • TypedDict 필수 — 타입 체크로 런타임 에러 방지
  • 스키마 변경 주의 — 체크포인트가 쌓인 후 스키마 변경하면 호환성 깨짐
  • 최소 필드 — 필요한 것만. 거대한 State는 직렬화/역직렬화 비용 증가

LangGraph 2.0 신기능 (2026)

  • Guardrail 노드 — 콘텐츠 필터링, 레이트 리밋, 감사 로깅이 1등 시민
  • MCP 통합 — Model Context Protocol로 외부 도구 표준화 연동
  • A2A 프로토콜 — Agent-to-Agent 메시지 큐 기반 크로스 프레임워크 통신
  • interrupt() API — 예외 기반이 아닌 명시적 중단점 + 구조화된 페이로드

8. LangGraph Platform & LangSmith

LangGraph Platform

에이전트 배포/관리 인프라

  • Cloud — LangChain 호스팅 (가장 쉬운 시작)
  • Hybrid — SaaS 컨트롤 플레인 + 자체 데이터 플레인
  • Self-Hosted — 전체 자체 인프라 (VPC 내)

내장 메모리, Studio 개발환경, API 제공

LangSmith

옵저버빌리티 & 디버깅

  • 모든 에이전트 호출 트레이스 시각화
  • 사이클 카운트, 인터럽트 포인트, 분기 결정 추적
  • 프로덕션 모니터링 + 알림
  • A/B 테스트, 평가 데이터셋

"순환 그래프를 옵저버빌리티 없이 디버깅하는 건 고통"
